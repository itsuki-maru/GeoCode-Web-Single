# プライバシーポリシー

このプライバシーポリシーは、GeoCode-Web-Single のソフトウェア本体、ソースコード、配布物、および本リポジトリ上の公開活動に関する個人情報等の取り扱い方針を説明するものです。

GeoCode-Web-Single は、利用者自身の PC または利用者が管理するサーバーで動作するアプリケーションです。作者が運営する中央サーバーへ、アプリ内データを自動送信する設計ではありません。

## 1. 作者が収集する情報

作者は、GeoCode-Web-Single のアプリケーション本体を通じて、利用者のマーカー、画像、動画、PDF、地図データ、アカウント情報、認証情報、位置情報、操作ログを自動的に収集しません。

ただし、GitHub などの公開サービス上で利用者が次の行為を行った場合、そのサービスの機能として情報が記録・公開されることがあります。

- Issue、Pull Request、Discussion、コメントの投稿
- リポジトリへの star、fork、watch
- セキュリティ報告や問い合わせ
- リリースファイルのダウンロードに伴う、GitHub 等のアクセスログ

これらの情報の取り扱いは、利用している各外部サービスのプライバシーポリシーにも従います。

## 2. アプリケーションが利用者環境で扱う情報

GeoCode-Web-Single は、利用者の環境内で次の情報を保存または処理します。

- 管理者および一般ユーザーのアカウント情報
- bcrypt でハッシュ化されたパスワード
- TOTP のシークレットおよび一時認証状態
- ログイン失敗回数、ロック状態、次回ログイン許可時刻
- レイヤ、マーカー、図形、地図上の緯度・経度
- Markdown 形式のメモ本文
- アップロードされた画像、PDF、動画、元ファイル名
- 画像サムネイルおよび動画 poster
- JSON インポート/エクスポートデータ
- 一時共有 URL、共有対象のスナップショット、共有パスワードのハッシュ、有効期限
- アプリ設定、JWT 用シークレット、Cookie 関連設定、タイルサーバー設定

通常の保存先は次のとおりです。

- 設定: `~/.geocode-web-single/geocode-web-single.env.json`
- SQLite DB: `~/.geocode-web-single/geocode-web.sqlite`
- アップロードファイル: `~/.geocode-web-single/images/`

これらのデータは、利用者またはアプリの運用者が管理する環境に保存されます。作者はこれらのデータへアクセスできません。

## 3. 外部送信とネットワーク通信

GeoCode-Web-Single は、通常、ローカル HTTP サーバーとブラウザ UI または Tauri WebView の間で通信します。

設定により外部タイルサーバー、リバースプロキシ、公開サーバー、共有 URL、GitHub Releases、その他の外部サービスを利用する場合、アクセス先のサーバーには IP アドレス、User-Agent、アクセス日時、リクエスト URL などが記録されることがあります。

国土地理院など外部の地図タイルを利用する場合、地図表示に必要なタイルリクエストが外部サービスへ送信される可能性があります。利用するタイルサーバーの規約およびプライバシーポリシーを確認してください。

## 4. Cookie と認証情報

GeoCode-Web-Single は、ログイン状態を管理するために JWT ベースのアクセストークンおよびリフレッシュトークンを HttpOnly Cookie として保存します。

Cookie は認証の維持、セッション管理、ログアウト、トークン更新のために利用されます。第三者広告やトラッキングを目的とした Cookie は、アプリ本体では使用しません。

HTTP 環境や外部公開環境で運用する場合は、`SECURE_COOKIE`、HTTPS、リバースプロキシ、公開範囲、Cookie 設定を適切に確認してください。

## 5. 共有 URL と公開範囲

サーバーとして GeoCode-Web-Single を起動し、ネットワークにアクセス可能な状態で 一時共有 URL 機能を使うと、選択したレイヤ、マーカー、図形、およびそれらに含まれる情報のスナップショットを、URL を知っている相手が閲覧できるようになります。

共有 URL には有効期限と任意の共有パスワードを設定できますが、共有対象に含めた内容、Markdown 本文、画像リンク、PDF、動画、位置情報などは、共有先から閲覧される可能性があります。

共有 URL を作成する前に、個人情報、機密情報、非公開の位置情報、第三者の権利を含むデータが含まれていないか確認してください。不要になった共有 URL は削除してください。

## 6. 画像・動画・PDF とメタデータ

画像はサーバー側で再エンコードされ、EXIF 等のメタデータを除去して保存されます。ただし、すべてのファイル形式、動画、PDF、本文、ファイル名、画像内に写り込んだ情報から個人情報や機密情報が完全に除去されることは保証しません。

アップロード前に、ファイル内容、ファイル名、写り込み、位置情報、文書プロパティ、PDF 内テキスト、動画音声などを確認してください。

## 7. 利用者・運用者の責任

GeoCode-Web-Single はローカルまたは自己管理環境で利用することを想定しています。データの管理、バックアップ、削除、アクセス権限、共有設定、ネットワーク公開設定、外部サービス利用設定は、利用者または運用者の責任で行ってください。

特に、サーバー単体モード、`0.0.0.0` での待ち受け、リバースプロキシ、HTTP 運用、共有 URL、ユーザー自己登録を有効化する場合は、公開範囲と認証設定を慎重に確認してください。

## 8. データの削除

アプリ内で作成したマーカー、レイヤ、図形、画像、共有 URL、ユーザー情報は、アプリの機能または SQLite DB と保存ディレクトリの管理によって削除できます。

完全な削除が必要な場合は、アプリのバックアップ、エクスポートファイル、配布済み共有 URL、ログ、OS や外部サービス側のキャッシュ、リリース成果物、GitHub 上の投稿なども確認してください。

## 9. 子どものプライバシー

GeoCode-Web-Single は、子どもを対象に個人情報を収集するサービスではありません。学校、家庭、団体などで利用する場合は、運用者が適用される法令、同意、管理方針に従ってください。

## 10. ポリシーの変更

本ポリシーは、機能追加、配布方法の変更、外部サービスの利用、法令や運用方針の変更に応じて更新されることがあります。重要な変更がある場合は、リポジトリ上の変更履歴、Release、または該当ドキュメントで案内します。

## 11. 問い合わせ

プライバシーに関する質問、修正依頼、削除依頼、セキュリティ上の懸念がある場合は、リポジトリの Issue、または必要に応じて非公開の連絡手段でメンテナーへ連絡してください。

脆弱性や公開すべきでない情報を含む内容は、公開 Issue ではなく [SECURITY.md](SECURITY.md) の案内に従ってください。
