## Version 2.1.0

- 図形描画機能を実装
  - 図形をユーザー共通ではなく `layer_id` 単位で保存・表示する
  - polygon / polyline / rectangle の作成・保存・削除に対応
  - 図形名の保存、ラベル表示、ラベルクリックによる名前変更に対応
  - 直前に削除した図形の復元に対応
- 共有マップ機能を強化
  - 共有リンクに任意のパスワードを設定できるように対応
  - 共有リンク作成時に図形を共有対象へ含める設定を追加
  - 共有ページで図形の表示・非表示切り替えと図形名表示に対応
  - パスワード保護された共有ページ用の入力画面を追加
  - 共有マップでも図形を `layer_id` ごとにグループ化し、マーカーと同じレイヤ切替に連動するよう更新
  - 図形を含む JSON エクスポート/インポートに対応
  - 旧形式のマーカーのみ JSON インポートとの後方互換を維持

## Version 2.0.8

- 動画コンテンツの poster 属性に対応
  - mp4 動画のアップデート時にフロントエンドで、poster 画像を作成し、同時に送信
  - バックエンドでは thumb ディレクトリに poster 画像を保存（XXXX-XXXX...XXX.mp4 と thumb/XXXX-XXXX...XXX.jpg） の UUID ファイル名は同一
  - 既存の動画データの poster を作成するスクリプトを用意
- アップロード時のプログレスモーダルの UX を向上（進捗がわかるようにした）
- 100MB 以上のファイルが選択された際に無効化する処理を実装（これまではサーバのみ）
- 100MB 以上のファイルがサーバに送信された場合にハンドラにおいても検査機構を実装
  1. Nginx で弾く
  2. axum の設定で弾く
  3. ハンドラでも検査（今回の実装）

## Version 1.0.0

GeoCode-Webをシングルバイナリで配布できるようにしたプロジェクト。小規模利用の際に、DockerあるいはPostgreSQLなどの環境構築不要で起動できる。データベースとしてSQLiteを使用。
