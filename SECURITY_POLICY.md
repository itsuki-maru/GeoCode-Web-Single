# セキュリティポリシー

## サポート対象バージョン

現時点では、メンテナンス対象はデフォルトブランチ上の最新版です。過去バージョンへのセキュリティ修正のバックポートは原則として保証していません。

## 脆弱性の報告方法

脆弱性の疑いがある場合は、公開 Issue を作成せず、まず GitHub の private reporting 機能が有効ならそれを使ってください。利用できない場合は、リポジトリ管理者の GitHub プロフィールに記載された連絡先から非公開で連絡してください。

連絡時に含めてほしい内容:

- 影響を受けるバージョン
- 再現手順
- 想定される影響
- 可能であれば暫定回避策

受領後は、内容確認、再現、修正方針の決定、公開の順で対応します。まだ修正されていない脆弱性の詳細は、修正版公開前に公開しないようお願いします。

## 運用時のセキュリティ注意点

- HTTP 運用では Cookie の `Secure` 属性に注意してください
- 共有 URL 機能を使う場合は、有効期限と共有範囲を必要最小限にしてください
- 管理者アカウントと TOTP を有効にし、配布済み端末のローカルアクセス権も管理してください
